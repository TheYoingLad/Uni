Exam

Exercise description

Create the project AdvJava-Sample-Exam and place all your files in it.

Part 1

Create the public, static field countryToBorderToSizes in class country.Traveller that contains a lambda. It doesn't take any arguments, and it starts processing the file country_borders.txt.

  • The file contains lines like this: Hungary;Austria:366;Croatia:329;Romania:443;Serbia:151;Slovakia:677;Slovenia:102;Ukraine:103
    • It means that Hungary has a 366 km long border with Austria, a 329 km one with Croatia etc.
  • Some lines look like this: Japan
    • It means that Japan is an isolated country with no land neighbours.
  • Using streams as best you can, do the following.
    • Take apart all lines along the ; characters.
    • Use Collectors.toMap.
      • The first parameter is the country. Make it the key of the Map.
      • All other bits are going to the value part of the Map (the second parameter of toMap), which is also going to be a Map, and is to be created with using Collectors.toMap on a stream once again.
        • Here, the key is simply the country's name.
        • The value is the border length between the two countries.
  • If there is a problem accessing the file, catch the exception, and return an empty HashMap.

Test the above functionality in method test of class country.TravellerTest.

  • Check that the resulting Map contains Hungary's neighbours with border lengths as they are seen on line 78 in the file.

Part 2

Create the public, static field getIsolatedCountries in class country.Traveller that contains a lambda. It takes the output of countryToBorderToSizes as a parameter, and returns a list of country names: those that have no borders with any other country.

  • The implementation of this must be a single (logically structured) stream.

Test the above functionality in method testIsolated of class country.TravellerTest.

  • The result is a list of 38 countries in total.
  • Here are all 38: Antigua and Barbuda;Australia;Bahamas;Bahrain;Barbados;Cape Verde;Comoros;Cuba;Dominica;Federated States of Micronesia;Fiji;Grenada;Iceland;Jamaica;Japan;Kiribati;Madagascar;Maldives;Malta;Marshall Islands;Mauritius;Nauru;New Zealand;Palau;Philippines;Saint Kitts and Nevis;Saint Lucia;Saint Vincent and the Grenadines;Samoa;Seychelles;Singapore;Solomon Islands;Sri Lanka;São Tomé and Príncipe;Tonga;Trinidad and Tobago;Tuvalu;Vanuatu
    • Your result does not need to be in alphabetical order.

Part 3

In the above, having to catch an exception is nasty. Let's avoid this as much as possible.

Create countryToBorderToSizes2 that is almost exactly a copy of countryToBorderToSizes, but instead of using a try block, the lambda's body is a call to ignoreException(..., ...).

  • ignoreException(..., ...) has a type parameter T takes two arguments and returns a type T value.
    • The first one's type is just like the lambda type of countryToBorderToSizes, but it uses T as its type argument.
    • The second one's type is similar, but it's a custom functional interface. Make it exactly like the lambda's type, but let its method throw any kind of Exception.
  • The body of ignoreException(..., ...) is like this.
    • First, invoke the lambda in the second argument. (As it doesn't take any parameters, so you can simply call it.)
    • If it doesn't throw an error, return the value that you get.
    • If an exception is thrown, print its stack trace, then invoke the first arg's lambda, and return that value.

Test this as well in TravellerTest.

Part 4

Now we are trying to travel the world so that we cross as long borders as we can. We'll use the genetic algorithm; our entities will be lists of country names; let's call them voyages.

First, invoke the countryToBorderToSizes lambda which reads the country borders from the file.

In main in class TravellerVoyage, call the genetic algorithm (the one created on the lab class; don't modify it) with the parameters 10, 100, 2, 10, 6 and the following ones.

  • createEntity: create a voyage of length between 2 and 6 with random countries
    • Create a static helper method getRndElem that returns a random element of a set. This works by making a stream of the set's elements, ignoring the first n (n is randomly chosen), then returning the one that you arrive at.
  • doCrossover: put together the first n countries of voyage #1 and the last m countries of voyage #2 where n and m are random (but at least 1)
  • doMutation: return a new voyage that consists of the same countries as the input, except that each country gets replaced by a random one with a 1% chance
  • fitnessValue: has two components: create two lambdas for it
    1. getGoodFitness: for each pairs of neighbouring countries, if they share a border, add the border length to a sum
      • add this with a negative sign, as this is "good" fitness
      • create the helper lambda neighbourFitness that takes a voyage and an index which computes the border length between countries index and index+1 in the voyage (or returns 0 if they're not neighbours)
    2. getPenalty: if there are n repeated countries in the voyage, add a value of 500 * n * (n+1)
      • this is "bad" fitness: we don't want to return to previously visited countries
      • create the helper lambda getRepeatCountryCount to compute this
        • hint: make a set out of all the countries in the voyage, and subtract that from the total country count

Finally, print the result and its fitness value: create displayVoyage that formats a voyage like the example below.

  • Use neighbourFitness, getRepeatCountryCount, getGoodFitness, getPenalty, and fitnessValue appropriately.
  • Its general format is: <neighbouring countries with fitness>, <N> countries, <N> repeats, total: <good fitness>+<fitness penalty>=<total fitness>

Example: if the best voyage looks like this:

[Saint Vincent and the Grenadines, Denmark, India, People's Republic of China, Russia, People's Republic of China, India, Kosovo, Liechtenstein, Austria]

... then the output of displayVoyage may look like this.

Saint Vincent and the Grenadines-Denmark:0, Denmark-India:0, India-People's Republic of China:-3380, People's Republic of China-Russia:-3645, Russia-People's Republic of China:-3645, People's Republic of China-India:-3380, India-Kosovo:0, Kosovo-Liechtenstein:0, Liechtenstein-Austria:-34, 10 countries, 2 repeats, total: -14084+3000=-11084

Part 5

Add testSymmetry to TravellerTest.

  • This method checks whether the result of countryToBorderToSizes is such that if country A's distance from country B is D, then country B's distance from country A is also D.
  • Using streams as best as you can, create a list of Executables that check this for each pair of countries in countryToBorderToSizes, and pass it to assertAll.
